Workflow cdev

Une seule commande pour tout gérer — de l'init à la session Claude Code interactive

01 Init
🧙 Wizard de configuration Une fois
cdev init
Guide interactif pas-à-pas : choix du mode d'auth (OAuth ou API key), saisie des identifiants PostgreSQL, build des images Docker, démarrage automatique de l'infra. Aucune connaissance Docker requise.
🔐 Authentification Claude Une fois
cdev auth
Détecte si un token existe déjà. Lance le flow OAuth claude.ai (lien à ouvrir dans le navigateur) ou utilise la clé API depuis .env. Token persisté dans le volume Docker claude-auth, partagé entre tous les projets.
02 Infra
🗄️ Démarrer l'infra partagée À chaque session
cdev start
Lance PostgreSQL et Redis sur dev-network. Vérifie que .env existe, sinon propose de lancer cdev init. Tous les projets partagent cette infra via leurs noms DNS internes.
📊 Vérifier l'état global Optionnel
cdev status
Affiche en un coup d'œil : état de l'infra (postgres, redis), statut de chaque projet (running / stopped), et statut de l'authentification Claude.
03 Projet
📦 Créer un projet Si besoin
cdev new mon-api
Wizard interactif : saisie du nom, port auto-incrémenté (modifiable), génération du Dockerfile + docker-compose.yml + code source. Propose de démarrer et d'ouvrir Claude Code immédiatement.
🚀 Démarrer un projet À chaque session
cdev up mon-api
Build et lance le container projet. Le code source vit dans un volume dédié mon-api-code:/app/src. Si aucun nom donné, affiche la liste interactive des projets disponibles.
04 Claude Code
🤖 Ouvrir la CLI interactive Usage quotidien
cdev code mon-api
Vérifie que le projet tourne (propose de le démarrer sinon), vérifie l'auth (propose cdev auth sinon), puis exécute docker exec --interactive --tty directement sur le binaire claude — TTY complet préservé.
💬 Travailler dans Claude Code Interactif
Toutes les fonctionnalités depuis /app/src : slash commands (/help, /clear, /model), lecture et édition de fichiers, accès PostgreSQL + Redis via les variables d'env, git intégré, sessions multi-turn continues.
🔄 Changer de projet Multi-projet
cdev code autre-projet
Chaque projet a son propre container et volume. Plusieurs sessions Claude Code peuvent tourner en parallèle dans des terminaux différents, toutes connectées à la même infra PostgreSQL / Redis.
05 Maintenance
⬆️ Mettre à jour Claude Code Occasionnel
cdev update
Rebuild les images Docker avec --no-cache pour récupérer la dernière version de @anthropic-ai/claude-code. Redémarre ensuite les projets avec cdev down + cdev up.
📋 Gérer les projets Quotidien
cdev list — voir tous les projets et leur statut   cdev logs <nom> — suivre les logs en temps réel   cdev down <nom> — arrêter proprement un projet   cdev stop — arrêter toute l'infra
🗑️ Réinitialiser tout Avec confirmation
cdev reset
Demande de taper supprimer pour confirmer. Arrête tous les containers et supprime tous les volumes (données PostgreSQL, tokens OAuth, code dans les volumes). Irréversible.
